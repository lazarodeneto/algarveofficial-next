┌─ ISSUE #1: QueryClient Cache Pollution (CRITICAL) ──────────────────────────┐
│                                                                              │
│ FILE: components/providers/AppProviders.tsx                                 │
│ PROBLEM: QueryClient created at module scope (singleton)                    │
│                                                                              │
│ IMPACT:                                                                      │
│   • Request #1 fetches data → cached in global QueryClient                 │
│   • Request #2 comes in → sees Request #1's cached data                    │
│   • Server renders with old data                                            │
│   • Client hydrates with old data                                           │
│   • But wait... old data is stale now!                                     │
│   • React detects hydration mismatch                                        │
│   • React discards server HTML                                              │
│   • Re-renders on client (defeats SSR!)                                    │
│   • curl gets minimal HTML (no content)                                    │
│                                                                              │
│ BEFORE:                                                                      │
│   const queryClient = new QueryClient({...});  // Created once at startup  │
│                                                                              │
│ AFTER:                                                                       │
│   const queryClient = useMemo(() => new QueryClient({...}), []);           │
│                                                                              │
│ WHY IT WORKS:                                                                │
│   ✅ useMemo creates fresh instance per component render                   │
│   ✅ Server: fresh instance per HTTP request                               │
│   ✅ Client: fresh instance per page navigation                            │
│   ✅ No cache pollution between requests                                    │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘

┌─ ISSUE #2: PublicSiteFrame Hydration Mismatch (CRITICAL) ─────────────────┐
│                                                                              │
│ FILE: components/layout/PublicSiteFrame.tsx                                │
│ PROBLEM: usePathname() returns undefined on server                         │
│                                                                              │
│ IMPACT:                                                                      │
│   1. Server tries to use usePathname()                                     │
│   2. Client hook not available on server → undefined                       │
│   3. Server renders sidebar with undefined pathname                        │
│   4. Client hydrates with actual pathname                                  │
│   5. Sidebar components render differently (useAuth, useRouter, etc)       │
│   6. React detects mismatch                                                 │
│   7. Discards server HTML, re-renders on client                            │
│   8. curl output has no server-rendered content                            │
│                                                                              │
│ BEFORE:                                                                      │
│   const pathname = usePathname();  // undefined on server                  │
│   if (shouldHideSidebar) return <>{children}</>;                           │
│   return <Sidebar />;  // Server might render this                         │
│                                                                              │
│ AFTER:                                                                       │
│   const [mounted, setMounted] = useState(false);                           │
│   useEffect(() => setMounted(true), []);                                   │
│   if (!mounted || shouldHideSidebar) return <>{children}</>;               │
│   return <Sidebar />;  // Only on client after hydration                   │
│                                                                              │
│ WHY IT WORKS:                                                                │
│   ✅ Server renders minimal HTML (just children)                           │
│   ✅ Client hydrates with matching HTML                                    │
│   ✅ After hydration, client adds sidebar (no mismatch)                    │
│   ✅ User sees sidebar appear smoothly                                     │
│                                                                              │

1. NEVER create singletons at module scope for SSR apps
   Use useMemo() to create fresh instances per render

2. ALWAYS defer client hooks to useEffect + useState
   Server can't use usePathname(), useRouter(), etc directly

3. ALWAYS test with curl to verify SSR
   If curl output doesn't have content, SSR is broken

4. ALWAYS add error handling to async operations
   One unhandled error can break the entire page

5. ALWAYS match server and client render output
   Any mismatch causes React to discard server HTML
